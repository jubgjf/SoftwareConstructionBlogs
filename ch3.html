# 软件构造过程与配置管理

# 传统软件开发过程模型

# 基本种类

  1. 线性过程
  2. 迭代过程

# 模型

  • 瀑布过程
    • 线性，非迭代
    • 整体推进
    • 管理简单
    • 无法适应需求变化
  • 增量过程
    • 线性，无迭代
    • 增量式，多个瀑布串行
    • 可以适应需求增加
  • V 字模型
    • 验证（verification）和确认（validation）
  • 原型过程
  • 螺旋模型

# SCM 和 VCS

# 基本概念

  • SCM：软件配置管理，追踪控制软件的变化
  • SCI：软件配置项：软件中发生变化的基本单元（如文件）
  • 基线：软件变化中的“稳定时刻”（如 release 版本）
  • CMDB：配置管理数据库：存储软件的各配置随时间变化的信息和基线

# 版本控制

版本号：

V 1.6.1.5

1.6 1 5
major minor patch

# 版本控制术语

仓库：基于 SCM 中的 CMDB HEAD：正在其上工作的版本

# VCS

  • 本地式：仓库存在本地
  • 集中式：仓库存在服务器
  • 分布式：仓库存在各个开发者的本地 + 服务器

# 基础

  • .git 文件夹：本地的 CMDB
